# Contributing to Weathercord
Thank you for your interest in contributing to Weathercord! There are many ways to contribute, and all are appreciated.

## Ways to Contribute

### Reporting Security Vulnerabilities
Please refer to Weathercord's [security policy](./SECURITY.md).

### Issues
If you discover a bug in Weathercord or have an idea for a new feature, check for duplicates before making an issue. If no duplicates exist, you may create a new issue.

### Pull Requests
If you would like to fix a bug or implement a new feature, you can choose an issue from the [issue tracker](https://github.com/raynepaws/weathercord/issues). Pull requests that fix issues with the https://github.com/raynepaws/weathercord/labels/help%20wanted label are particularly appreciated. Pull requests making changes that don't resolve any particular issue(s) have a much lower chance of being merged, and will probably end up being a waste of your time. It is advised to only make pull requests that resolve issues with the https://github.com/raynepaws/weathercord/labels/approved label. New contributors should consider checking the https://github.com/raynepaws/wearhercord/labels/good%20first%20issue label.

## Contribution Guidelines
- Please write descriptive and accurate commit messages.
- If your changes can't be summed up in a short commit message, the commit is too large. Commit often.
